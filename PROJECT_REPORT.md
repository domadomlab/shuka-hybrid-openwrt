# Подробный отчет о разработке проекта Shuka Hybrid Suite v2.5.3 LTS

**Дата завершения:** 14 Июня 2026 г.
**Целевая платформа:** OpenWrt 21.02-SNAPSHOT (GL.iNet GL-MT3600BE / aarch64_cortex-a53)
**Версия Ядра:** Linux 5.4.281

---

## 1. Концепция и цели проекта
Проект **Shuka Hybrid Suite** был разработан с целью создания ультимативного VPN-комбайна для домашних роутеров. Главная задача заключалась в объединении двух мощных технологий обхода блокировок в едином, удобном веб-интерфейсе:
1. **AmneziaWG (AWG 2.0)** — для использования персональных конфигураций с обфускацией (Junk packets, Magic bytes).
2. **Sing-box** — для работы с современными подписками (VLESS, Shadowsocks, Trojan) с поддержкой протоколов Reality и XTLS-Vision.

Система должна была стать автономной (автозапуск), отказоустойчивой (защита от потери интернета) и интуитивно понятной для пользователя.

---

## 2. Применяемый стек технологий

### ⚙️ Сетевые движки и Протоколы
*   **Нативный модуль ядра AmneziaWG (`amneziawg.ko`):** В ходе разработки был осуществлен переход с пользовательского драйвера `amneziawg-go` на нативную поддержку ядра. Это обеспечило нулевой оверхед по CPU и максимальную пропускную способность. Для управления используется системная утилита `/usr/bin/awg`.
*   **Sing-box (v1.13.x):** Использован как мощный прокси-клиент и маршрутизатор. Внедрена поддержка актуального стандарта конфигурации (v1.13+), включая отказ от устаревших правил `dns outbound`.
*   **Smart DNS (DoH):** Весь теневой трафик направляется через Google DNS по зашифрованному HTTPS каналу (`https://8.8.8.8/dns-query`), что исключает возможность перехвата или подмены DNS-запросов провайдером.

### 💻 Программная логика (Бэкенд)
*   **Python 3.9:** Использован для написания ядра системы — `shuka_manager.py`. Менеджер отвечает за:
    *   Парсинг подписок по URL.
    *   Декодирование Base64 строк (с автоматическим восстановлением отсутствующего padding'а).
    *   Разбор сложных ссылок VLESS/SS с помощью регулярных выражений (RegEx).
    *   Динамическую генерацию `config.json` для Sing-box на основе эталонного шаблона.
    *   Управление состояниями (state.json).
*   **Bash & iptables:** Написаны скрипты для управления сетью (`start_shuka.sh`, `internet-protection.sh`). Реализовано динамическое создание `tun` интерфейсов, автоматическое прописывание MASQUERADE NAT и правил FORWARD.

### 🌐 Веб-интерфейс (Фронтенд)
*   **LuCI (Lua 5.1):** Написан кастомный контроллер `shuka_hybrid.lua`. Реализованы точки входа (API) для управления всеми функциями (start, stop, sync, delete) и получения потоковых данных о состоянии сети.
*   **Vanilla JavaScript (ES5/AJAX):** Интерфейс пользователя переписан на классический `XMLHttpRequest`. Это сделало систему «бронебойной» и совместимой с любыми, даже устаревшими браузерами. Внедрен механизм мгновенного отклика (Instant UI Feedback) без ожидания ответа от сервера.
*   **HTML5 / CSS3:** Создан современный, адаптивный дизайн (Dark Mode) с анимациями загрузки (Spinner) и цветовой индикацией статусов сети.

## 3. Решенные критические проблемы

В ходе разработки и глубокого дебаггинга были устранены следующие сложные инженерные вызовы:

1. **Конфликт библиотек Sing-box (musl vs glibc):**
   * *Проблема:* Роутеры OpenWrt используют библиотеку `musl`, а бинарный файл Sing-box требовал `glibc`.
   * *Решение:* Написан автоматический патч, создающий симлинк `/lib/ld-linux-aarch64.so.1` на `libc.so`, что позволило запускать бинарники без перекомпиляции прошивки.

2. **Зависание интерфейсов при падении туннеля:**
   * *Проблема:* Если сервер VPN переставал отвечать, роутер терял интернет намертво.
   * *Решение:* Разработан `internet-protection.sh` — агрессивный Watchdog. Он пингует независимые узлы и, при обрыве связи на 10 секунд, выполняет глубокую очистку таблиц маршрутизации (`ip route flush cache`) и правил `iptables`, возвращая пользователю обычный интернет. Для предотвращения ложных срабатываний при запуске VPN внедрен механизм «умной паузы» (lock-файлы).

3. **Ошибки парсинга Base64 и Unicode:**
   * *Проблема:* При скачивании подписок серверы, содержащие эмодзи (🔥, 🇫🇷) или кириллицу в названиях, ломали конфигурационный файл.
   * *Решение:* В Python-скрипт внедрена принудительная обработка `utf-8` и параметр `ensure_ascii=False` при записи JSON, что сохранило человекочитаемый вид конфигов.

4. **Асинхронность веб-интерфейса и Lua:**
   * *Проблема:* В старых версиях Lua цепочки команд вида `io.open():write():close()` вызывали краш сервера `uhttpd`.
   * *Решение:* Код переписан с соблюдением строгих стандартов безопасности памяти Lua. Анимация интерфейса (UPDATING...) отвязана от выполнения скриптов и привязана к наличию временного lock-файла (`/tmp/shuka_syncing`), обеспечивая 100% надежную визуализацию процессов.

5. **Белые списки (Bypass Routing):**
   * *Проблема:* Российские сервисы не открывались из-за конфликта DNS или попыток маршрутизации через зарубежные узлы.
   * *Решение:* В структуру Sing-box внедрен `dns-direct`. Домены `.ru` и специфичные сервисы (Госуслуги, Яндекс, ВК) перенаправляются в системный интерфейс (`outbound: direct`), минуя туннель.

## 4. Архитектура и Файловая Система

*   `/usr/bin/shuka_manager.py` — Мозг системы (управление и парсинг).
*   `/usr/lib/lua/luci/controller/shuka_hybrid.lua` — API и графический интерфейс.
*   `/etc/sing-box/config.json.template` — Эталонный шаблон маршрутизации.
*   `/etc/amneziawg/profiles/` — Хранилище загруженных пользователем файлов `.conf`.
*   `/usr/bin/internet-protection.sh` — Служба защиты и сброса маршрутов.
*   `/etc/init.d/shuka-boot` — Служба автозапуска последнего активного профиля при ребуте.

## 5. Итоги
Проект успешно доведен до статуса **v2.5.3 LTS Final**. Система запакована в единый установочный пакет `luci-app-shuka-hybrid_2.5.3_aarch64.ipk`, который самостоятельно устанавливает все Python-зависимости, настраивает права и интегрируется в систему роутера.

Открытый и анонимизированный исходный код (без ключей и паролей) опубликован на GitHub для дальнейшей поддержки.
